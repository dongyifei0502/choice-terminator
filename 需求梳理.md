## 1. 项目背景

「选择困难终结」是一款帮助用户在多个选项间快速决策的 **Web 全栈应用**。当用户面对多个选项无法决定时（如选餐厅、选活动、选商品），通过多维度提问 + 加权打分，用数据支撑的方式给出推荐结果。

**一句话总结**：面对多个选项无法决定的人，打开网页花 1 分钟走完流程，得到一个数据支撑的答案。

## 5. 业务流程

```
用户打开网页 → 选择场景（餐饮/购物/旅行/通用）
  → 输入选项 → 开始决策
  → 逐题打分（最多 5 题，每题对每个选项打 1-5 分）
  → 加权计算总分
  → 展示推荐结果（🏆 + 柱状图 + 理由 + 明细）
  → 保存到历史（登录后云端同步）
  → 可回看历史 / 导出 / 重新开始
```

## 6. 数据与存储

| 数据类型 | 存储方式 | 使用者 |
|----------|----------|--------|
| 问题库 | SQLite（云端）+ localStorage（本地兜底） | 管理员读写系统题，用户可建自选题 |
| 历史记录 | SQLite（云端）+ localStorage（本地兜底） | 按用户隔离，登录后云端同步 |
| 用户数据 | SQLite users 表 | 管理员可查看用户列表 |
| 配置文件 | data/config.json | 程序自动读写 |

### 数据安全

- 密码使用 SHA-256（盐值）哈希存储，不明文
- JWT 令牌使用 HMAC-SHA256 签名，72 小时过期
- 登录用户数据云端隔离（每人只能看自己的历史）
- 被封禁用户无法登录

## 7. 非功能需求

### 7.1 性能

| 指标 | 实际表现 |
|------|----------|
| 首页加载 | < 1 秒（纯静态 HTML + CDN Chart.js） |
| 页面切换 | 即时（SPA display 显隐，无网络请求） |
| 打分计算 | < 10ms（纯前端数组运算） |
| API 响应 | < 50ms（SQLite + Flask 本地查询） |

### 7.2 安全性

| 维度 | 措施 |
|------|------|
| 密码存储 | SHA-256 哈希 + 固定盐值 |
| 认证 | HMAC-SHA256 手写 JWT，72h 过期 |
| 数据隔离 | 历史记录按 user_id 隔离 |
| 封禁机制 | 被封禁用户返回 403，禁止登录 |
| XSS | escapeHTML() 转义用户输入 |
| CORS | 手写 after_request 中间件 |

### 7.3 兼容性

| 环境 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Edge 90+, Firefox 90+ |
| 屏幕 | ≥ 1280×720，响应式（700px 断点） |
| 部署 | PythonAnywhere / 本地 python app.py / 直接打开 index.html |

### 7.4 可用性

- 首次访问无需登录，直接可用决策功能
- 登录后解锁云端同步 + 问题管理
- 所有操作有明确视觉反馈（hover/active/focus）
- 错误状态有明确提示
- 平局时有 🎲 随机选项避免卡住

## 11. 风险与解决方案

| 原风险 | 实际解决方案 |
|--------|-------------|
| 问题库设计不合理 | 预设 20 道题（4 场景 × 5 题），支持自定义 |
| 全部选项同分无法决策 | 随机选择 + 🎲 换一个按钮 |
| 数据丢失 | 云端同步 + localStorage 兜底 + JSON/CSV 导出 |
| 非管理员无法管问题 | 所有登录用户均可管自有问题，管理员管系统题 |
| 部署门槛高 | PythonAnywhere 免费版，仅依赖 Flask |
