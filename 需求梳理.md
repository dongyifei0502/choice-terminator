# 需求梳理 — 选择困难终结

## 1. 项目背景

一款帮助用户在多个选项间快速决策的工具。当用户面对多个选项无法决定时（如选餐厅、选活动、选商品），通过多维度提问 + 加权打分，用数据支撑的方式给出推荐结果。

**一句话总结**：任何面对多个选项无法决定的人，打开软件花 1 分钟走完流程，得到一个数据支撑的答案。

## 2. 目标用户

| 用户角色   | 场景                     | 解决问题                                  |
|------------|--------------------------|-------------------------------------------|
| 普通用户 A | 午饭时间，不知道吃哪家   | 在火锅/沙拉/快餐中快速决策                 |
| 普通用户 B | 周末纠结去哪玩           | 在爬山/看电影/逛商场/宅家中选择            |
| 普通用户 C | 网购选择困难             | 在几个相似商品间对比决策                   |
| 管理员     | 系统维护者               | 定期更新问题库，让提问更贴合实际           |

## 3. 核心功能

### 3.1 功能概览（01 功能概述）

- **选项输入**：用户可输入 2-6 个选项（如"火锅、烧烤、日料"），支持逗号/空格分隔
- **多轮提问**：系统依次提问 N 个维度的问题（如"你对价格敏感吗？"）
- **打分机制**：每轮给每个选项打分（1-5 分，1=非常不符合，5=非常符合）
- **权重系统**：每个问题有独立权重（1-3 倍），体现问题重要性
- **自动计算**：累计加权得分，选出最优选项
- **推荐理由**：基于得分差异，生成自然语言的推荐解释
- **可视化**：用柱状图展示各选项最终得分对比
- **历史记录**：保存最近 10 次决策，可查看过往选择
- **管理员功能**：可增删改查问题库，调整问题权重
- **导出功能**：可将历史决策导出为 JSON 或 CSV 文件

### 3.2 优先级划分

| 优先级 | 功能模块     | 功能描述                                           |
|--------|-------------|---------------------------------------------------|
| P0     | 选项输入     | 输入 2-6 个选项，支持逗号/空格分隔                  |
| P0     | 多轮提问     | 逐题展示，含打分滑块（1-5 分）                      |
| P0     | 加权计算     | 每道题独立权重 × 各选项得分，累加得出最终分         |
| P0     | 结果展示     | 推荐选项高亮 + 各选项柱状图对比                     |
| P1     | 推荐理由     | 自然语言解释为什么推荐该选项                       |
| P1     | 可视化       | 柱状图展示最终得分对比                             |
| P1     | 历史记录     | 最近 10 次决策，点击可查看详情                      |
| P2     | 管理员功能   | 问题库增删改查 + 权重调整                           |
| P2     | 导出功能     | JSON / CSV 导出历史决策                            |

## 4. 核心页面（03 核心页面）

共 **5 个主页面**：

| 页面名称 | 承载功能                                                                 |
|----------|--------------------------------------------------------------------------|
| 首页     | 标题展示、选项输入框（支持逗号/空格分隔）、开始决策按钮、历史记录入口、管理员入口 |
| 问答页   | 当前问题文本、每个选项打分滑块（1-5 分）、权重倍数显示、进度条（第 X 题/共 N 题）、下一题按钮 |
| 结果页   | 推荐选项（大字体高亮）、推荐理由（2-3 句话）、各选项得分柱状图、重新开始按钮、保存到历史按钮 |
| 历史页   | 最近 10 次决策（时间戳、选项列表、推荐结果），点击可查看详情（重新打开结果页） |
| 管理页   | 问题列表（展示所有问题/权重/是否启用）、添加/编辑/删除问题、调整权重滑块、保存修改 |

## 5. 业务流程（02 用户场景）

```
用户输入选项 → 依次回答 N 道题（每题给每个选项打 1-5 分）
→ 系统按权重计算加权总分 → 展示推荐结果 + 柱状图 + 推荐理由
→ 用户可选择保存到历史 或 重新开始
```

## 6. 数据与存储（04 数据和使用）

| 数据类型   | 存储文件              | 使用者               | 备份策略                               |
|------------|----------------------|----------------------|----------------------------------------|
| 问题库     | `questions.json`     | 管理员读写，用户只读 | 程序启动时自动备份为 `questions_backup.json` |
| 历史决策   | `history.json`       | 普通用户读写         | 每次新增记录时自动备份（保留最近 5 份）  |
| 用户配置   | `config.json`        | 程序自动读写         | 不需要备份，可重置默认                  |

**数据安全考虑**：
- 所有数据本地存储，不上传任何服务器
- 支持「导出全部数据」为 ZIP 包
- 支持「清空历史」功能

## 7. 非功能需求

- **平台**：Windows 桌面应用
- **性能**：打分计算实时响应，页面切换无感知延迟
- **安全性**：纯本地存储，无网络请求
- **可用性**：界面简洁，1 分钟内完成完整决策流程
- **数据持久化**：JSON 文件存储，程序关闭后数据不丢失

## 11. 风险与开放问题

- **选项数量上限**：当前定为 2-6 个，是否需要支持更多？
- **问题数量**：每次决策提问多少题？每题都要对所有选项打分，题太多影响体验
- **推荐理由生成**：是基于规则模板还是接入 AI？当前倾向规则模板
- **技术栈未定**：需要确认桌面应用的技术方案
- **图表库选型**：柱状图实现方案待定
