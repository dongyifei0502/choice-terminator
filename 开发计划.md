## 阶段五：云端部署

### 部署平台：PythonAnywhere（免费版）

**步骤**：

1. 用户注册 PythonAnywhere → 创建 Manual Config Web App → Python 3.10
2. API 上传 31 个项目文件
3. 配置 WSGI：`/var/www/dariaaaa_pythonanywhere_com_wsgi.py`

**遇到的问题及解决**：

| 问题 | 解决方案 |
|------|----------|
| Render 需要信用卡 | 换 PythonAnywhere |
| WSGI 目录不存在 | API 创建 var/www/ 目录 |
| init_db 不调用（WSGI 模式） | 移到模块顶层自动执行 |
| sql/ 未上传 | API 补充上传 |
| 静态文件丢失 | 重新批量上传 |
| flask-cors/pyjwt/bcrypt 不可安装 | 手写替代：CORS after_request + SHA-256 + HMAC |

**最终状态**：
- 线上：https://dariaaaa.pythonanywhere.com
- 依赖：仅 Flask（PythonAnywhere 预装）
- 数据库：SQLite（首次请求自动创建）

## 附录 A：项目技术栈总览

### A.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端标记 | HTML5 | 单文件 6 页面 SPA |
| 前端样式 | CSS3 | CSS 变量 + 暖色手绘 + 响应式（700px） |
| 前端脚本 | Vanilla JS (ES6+) | 无框架/TypeScript/构建工具 |
| 前端图表 | Chart.js v4 | CDN 引入 |
| 前端存储 | localStorage | 离线兜底 |
| 后端框架 | Flask 3.x | Python WSGI |
| 数据库 | SQLite | 零配置单文件 |
| 认证 | SHA-256 + HMAC | 手写替代 bcrypt + PyJWT |
| 部署 | PythonAnywhere | 免费版，Python 3.10 |

### A.2 不使用的技术

| 不使用的 | 原因 |
|----------|------|
| Node.js/npm/webpack | 纯静态 HTML + Python 后端 |
| React/Vue/Angular | 6 页面，Vanilla JS 足够 |
| TypeScript | JSDoc 注释即可 |
| bcrypt/PyJWT/flask-cors | PythonAnywhere 免费版限制 |

### A.3 JS 加载顺序

```
constants.js → store.js → parser.js → sanitizer.js
→ questionService.js → historyService.js
→ scoreEngine.js → reasonGenerator.js → exportService.js
→ apiService.js
→ progressBar.js → slider.js → barChart.js → modal.js
```

## 附录 C：关键数据流

### C.1 完整决策数据流

```
用户选择场景 → 输入选项
  → parser.js 解析 → Store.quiz.options
  → 点击"开始决策"
  → QuestionService.selectQuestions(category) 按权重降序选题（最多5题）
  → 问答页逐题展示 + 用户拖动滑块打分
  → Store.quiz.scores = { q001: {火锅:2, 烧烤:3, 日料:5}, ... }
  → 最后一题 → 点击"查看结果"
  → ScoreEngine.run()
    → calculateFinalScores → {火锅:19, 烧烤:21, 日料:22}
    → selectWinner → 随机平局处理 → {winner: "日料", tiedOptions: [...], isTie: false}
  → ReasonGenerator.generate(...) → "在3个选项中..."
  → 结果页渲染：推荐pill + 理由 + 柱状图(Chart.js) + 明细表
  → 用户点击"保存到历史"
    → 已登录 → ApiService.saveHistory() → 云端 SQLite
    → 未登录 → HistoryService.save() → localStorage
```

### C.2 存储层级

```
云端（优先）:
  SQLite — 登录用户的历史、问题、用户数据
  ↓ API 调用失败时降级

本地（兜底）:
  localStorage — 历史记录、问题库缓存、JWT 令牌
  ↓ localStorage 不可用时降级

静态文件（兜底）:
  data/questions.json — 默认 20 道题（fetch 加载）
  data/config.json — 默认配置
```
