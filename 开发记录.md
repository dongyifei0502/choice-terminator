## 六、阶段五：云端部署到 PythonAnywhere

### Render 尝试（失败）

> **问题**：Render API 返回 `Payment information is required`，免费套餐也需要信用卡。
>
> **解决**：用户选择不用信用卡，转 PythonAnywhere。

### PythonAnywhere 部署过程

**步骤 1**：用户在 pythonanywhere.com 注册 → 创建 Manual Config Web App → Python 3.10

**步骤 2**：通过 PythonAnywhere REST API 上传全部 31 个项目文件

**步骤 3**：配置 WSGI 文件 `/var/www/dariaaaa_pythonanywhere_com_wsgi.py`

**遇到的关键问题及解决**：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| WSGI 文件路径找不到 | API 上传 200 但文件未保存；`/var/www/` 目录不存在 | 先创建 var/、var/www/ 目录，再上传 |
| 网站 HTTP 500 | `init_db()` 只在 `if __name__ == '__main__'` 中调用，WSGI 模式不触发 | 将 `init_db()` 移到模块顶层自动执行 |
| 网站 HTTP 500（二次） | `sql/` 目录未上传，`init_db()` 读不到 schema.sql | API 上传 sql/schema.sql 和 sql/seed.sql |
| 网站 HTTP 404 | 静态文件（index.html/css/js）未上传成功 | 重新批量上传所有静态文件 |
| 网站 HTTP 500（API 路由） | `flask-cors` `pyjwt` `bcrypt` 未安装且免费版无法 pip install | **精简依赖**：手写 CORS 中间件、SHA-256+HMAC 替代 bcrypt+PyJWT，最终只依赖 Flask |

### 最终状态

- **上线地址**：`https://dariaaaa.pythonanywhere.com` ✅
- **依赖**：仅 `flask`（PythonAnywhere 预装）
- **数据库**：SQLite，首次请求自动创建
- **管理员**：admin / admin123（SHA-256 哈希存储）

## 七、项目技术栈总览

### 前端

| 层级 | 技术 | 说明 |
|------|------|------|
| 标记语言 | HTML5 | 单文件 5 页面 SPA 结构（section 显隐切换） |
| 样式 | CSS3 | CSS 变量系统、暖色手绘风、粗黑描边、响应式（700px 断点） |
| 脚本 | Vanilla JS (ES6+) | 无框架、无 TypeScript、无构建工具；IIFE 模块模式 |
| 图表 | Chart.js v4 | CDN 引入，柱状图可视化 |
| 存储 | localStorage | 离线兜底，云端优先 |
| 路由 | 手动页面切换 | `Router.navigate()` — display 控制显隐 + 页面 init 函数 |

### 后端

| 层级 | 技术 | 说明 |
|------|------|------|
| Web 框架 | Flask 3.x | Python 轻量 WSGI 框架 |
| 数据库 | SQLite | 零配置单文件数据库 |
| 认证 | SHA-256 + HMAC | 手写替代 bcrypt + PyJWT（零外部依赖） |
| CORS | 手写 after_request | 替代 flask-cors |
| 密码存储 | hashlib.sha256 | 固定盐值哈希 |

### 部署

| 层级 | 技术 | 说明 |
|------|------|------|
| 平台 | PythonAnywhere | 免费版，Python 3.10 + Flask 预装 |
| 配置 | Manual Config WSGI | `/var/www/dariaaaa_pythonanywhere_com_wsgi.py` |
| 文件上传 | REST API | PythonAnywhere Files API（multipart/form-data） |

### 不使用的技术

| 不使用的 | 原因 |
|----------|------|
| Node.js / npm | 纯静态前端 + Python 后端 |
| React / Vue / Angular | 5 页面规模小，Vanilla JS 足够 |
| webpack / vite / parcel | 无构建需求 |
| TypeScript | 保持简单，JSDoc 注释即可 |
| bcrypt / PyJWT / flask-cors | PythonAnywhere 免费版不可安装，手写替代 |

## 九、API 接口文档

### 认证

| 方法 | 端点 | 鉴权 | 说明 |
|------|------|:---:|------|
| POST | `/api/auth/register` | 无 | 注册 `{username, password}` → `{token, user}` |
| POST | `/api/auth/login` | 无 | 登录 → `{token, user}`（被封禁用户返回 403） |
| GET | `/api/auth/me` | Bearer | 当前用户信息 |
| GET | `/api/auth/admin/users` | Admin | 所有用户列表 |
| PUT | `/api/auth/admin/users/<id>/ban` | Admin | 封禁用户 |
| PUT | `/api/auth/admin/users/<id>/unban` | Admin | 解封用户 |

### 问题库

| 方法 | 端点 | 鉴权 | 说明 |
|------|------|:---:|------|
| GET | `/api/questions` | Bearer | 系统题 + 当前用户自建题 |
| POST | `/api/questions` | Bearer | 新增自建题 `{category, text, weight}` |
| PUT | `/api/questions/<id>` | Bearer | 编辑（自己的/管理员全部） |
| PUT | `/api/questions/batch` | Admin | 批量保存系统题 |
| DELETE | `/api/questions/<id>` | Bearer | 删除（自己的/管理员全部） |

### 历史记录

| 方法 | 端点 | 鉴权 | 说明 |
|------|------|:---:|------|
| GET | `/api/history` | Bearer | 当前用户历史（倒序） |
| POST | `/api/history` | Bearer | 保存一条 `{id, options, scores, ...}` |
| DELETE | `/api/history/<id>` | Bearer | 单条删除 |
| DELETE | `/api/history/clear` | Bearer | 清空当前用户历史 |

### 配置

| 方法 | 端点 | 鉴权 | 说明 |
|------|------|:---:|------|
| GET | `/api/config` | 无 | 系统配置 |
