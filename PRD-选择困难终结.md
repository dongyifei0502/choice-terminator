## 1. 产品概述

### 1.1 产品定位

「选择困难终结」是一款**纯本地运行的 Web 单页应用**，帮助用户在多个选项间通过多维度加权打分的方式做出数据支撑的决策，彻底告别"选择困难症"。

### 1.2 一句话描述

面对多个选项无法决定时，花 1 分钟走完提问打分流程，得到一个有理有据的推荐答案。

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 选项太多，不知怎么比 | 系统引导从多个维度逐一打分，强制结构化思考 |
| 决策凭感觉，选完后悔 | 加权计分 + 推荐理由，让决策有数据支撑 |
| 反复纠结，浪费时间 | 1 分钟内完成，给出明确结论 |

### 1.4 典型决策流程

```
首页输入选项 → 问答页逐题打分 → 结果页展示推荐 + 柱状图 + 理由 → 保存/重来
```

一次完整决策耗时约 **30-60 秒**（按 5 题、3 个选项估算）。

## 2. 用户分析

### 2.1 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| **普通用户** | 任何面临多选项决策困难的人 | 快速输入 → 打分 → 得到推荐 |
| **管理员** | 维护问题库的人（可能是同一用户） | 增删改查问题、调整权重、启停用题目 |

### 2.2 典型使用场景

| 场景 | 示例选项 | 可能涉及的问题维度 |

| 选旅游目的地 | 成都、大理、三亚、西安 | 预算、旅行天数、风景偏好、美食重要度 |

### 2.3 用户权限矩阵

## 3. 功能需求详述

### 3.1 选项输入（P0）

**功能描述**：用户在首页输入待决策的多个选项，作为后续打分的对象。

**交互规则**：
- 输入框支持**逗号**（中英文均可 `,` `，`）和**空格**作为分隔符
- 最少输入 **2 个**选项，最多 **10 个**选项
- 实时解析：输入时自动识别分隔符，在输入框下方以标签（Tag）形式预览已解析的选项
- 不足 2 个选项时，「开始决策」按钮置灰不可点击
- 超过 10 个时，提示"最多支持 10 个选项，请精简"

**边界处理**：
- 连续分隔符（如"火锅,,烧烤"）自动过滤空项
- 首尾空格自动 trim
- 单选项内允许包含空格（如"番茄 牛腩面"），不做二次拆分

**数据结构**（输入阶段）：

```js
// 用户输入字符串
input: "火锅, 烧烤, 日料"

// 解析后的选项数组
options: ["火锅", "烧烤", "日料"]
```

---

### 3.2 多轮提问与打分（P0）

**功能描述**：系统从问题库中按顺序逐一展示问题，用户为每个选项在该问题维度下打分。

**交互规则**：
- 每次决策提问数量 = 管理员启用的问题数，但 **最多 5 题**
  - 若启用问题数 > 5，取权重最高的 5 题
  - 若启用问题数 ≤ 5，全部使用
- 每题展示内容：
  - 问题文本（如"你对价格敏感吗？"）
  - 每题权重标签（如"权重：×2"，以徽标 Badge 展示）
  - 每个选项的**打分滑块**（range slider），范围 1-5 分，默认值 3
  - 滑块两端标注：`1 = 非常不符合` `5 = 非常符合`
  - 进度条：`第 2 题 / 共 5 题`
- 点击「下一题」进入下一题；最后一题按钮文案变为「查看结果」
- 支持**回退上一题**修改打分（保留已打分数）
- 必须所有选项都打过至少一次交互（即滑块被拖动或点击），才能进入下一题（默认值 3 视为已作答，允许不修改直接提交）

**滑块刻度**：

| 分数 | 含义 |
|:---:|------|
| 1 | 非常不符合 |
| 2 | 比较不符合 |
| 3 | 一般 |
| 4 | 比较符合 |
| 5 | 非常符合 |

---

### 3.3 加权计算（P0）

**功能描述**：汇总所有题目中的所有选项打分，乘以各自问题的权重，累加得出每个选项的最终得分。

**计算时机**：用户点击最后一题的「查看结果」时触发。

**计算公式**：

```
选项X的最终得分 = Σ(问题i的权重 × 选项X在问题i上的得分)
                  i=1..n
```

**计算示例**（3 个选项，3 道题）：

| 问题（权重） | 火锅得分 | 烧烤得分 | 沙拉得分 |
|-------------|:---:|:---:|:---:|
| 价格敏感吗？（×3） | 2 | 3 | 5 |
| 口味重要吗？（×2） | 5 | 4 | 2 |
| 用餐环境重要吗？（×1） | 3 | 4 | 3 |

```
火锅总分 = 3×2 + 2×5 + 1×3 = 6 + 10 + 3 = 19
烧烤总分 = 3×3 + 2×4 + 1×4 = 9 + 8 + 4 = 21
沙拉总分 = 3×5 + 2×2 + 1×3 = 15 + 4 + 3 = 22
→ 推荐：沙拉
```

**平局处理**：若得分相同，取排序靠前的（即选项中排最前面的），并在推荐理由中注明"与 XX 得分相同，根据输入顺序推荐"。

---

### 3.4 结果展示（P0）

**功能描述**：结果页是决策流程的终点，展示推荐结论、得分详情和可视化图表。

**页面元素**：
- **推荐选项**：大字高亮显示（如"🏆 推荐：沙拉"），字号为页面最大
- **最终得分**：推荐选项的加权总分
- **推荐理由**：2-3 句自然语言解释（详见 7. 推荐理由生成）
- **柱状图**：所有选项的得分对比图，推荐选项柱子高亮/加粗颜色
- **得分明细表**（可折叠）：每题每个选项的原始分 × 权重的明细
- **操作按钮**：
  - 「重新开始」→ 回到首页
  - 「保存到历史」→ 存入 history.json

---

### 3.5 可视化（P1）

**功能描述**：用柱状图展示各选项最终得分对比。

**图表规格**：
- **类型**：垂直柱状图（Column Chart）
- **X 轴**：选项名称
- **Y 轴**：加权总分
- **推荐项**：柱子使用高亮色（如金色/橙色），其余使用默认色（如蓝色）
- **柱顶**：标注具体分数数值
- **实现方案**：使用 Chart.js 或 ECharts（纯前端图表库，无后端依赖）

**视觉效果**：
- 柱子高度差能直观反映选项间的差距
- 差距较小时（< 5%），图表下方标注"差距较小，可根据个人偏好决定"

---

### 3.6 推荐理由生成（P1）

**功能描述**：基于得分数据自动生成 2-3 句自然语言解释。

**双模式支持**：

| 模式 | 实现方式 | 优点 | 默认 |
|:---:|------|------|:---:|
| 规则模板 | 基于得分数据填充预设模板 | 离线可用，响应即时，稳定可控 | ✅ |
| AI 增强 | 调用 LLM API 生成更自然的理由 | 语言更生动，能捕捉微妙差异 | ❌ |

**规则模板逻辑（默认模式）**：

```
理由由以下部分组成，按优先级组装：

1. "在 N 个选项中，【选项X】综合得分最高（XX 分）"

2. 识别推荐项得分最高的维度（加一句）：
   - 若某维度得分 = 5：  "尤其在「【维度名】」方面表现突出。"
   - 若某维度得分 ≥ 4：  "在「【维度名】」方面表现较好。"

3. 如果与第二名分差 ≥ 10%：
   - "相比第二名【选项Y】有明显优势（领先 X 分）。"
   - 否则：  "与第二名【选项Y】差距不大，可凭直觉微调。"

4. 如果权重最高的题目上推荐项得分最高：
   - "且在你最看重的「【最高权重题目】」维度上也表现最佳。"
```

**AI 增强模式（需配置）**：
- 用户可在设置中填入 API Key（支持 OpenAI 兼容接口）
- 发送时仅传选项名 + 得分 + 权重维度的结构化数据，不传个人隐私
- AI 不可用时自动降级为规则模板

---

### 3.7 历史记录（P1）

**功能描述**：保存用户最近的决策记录，支持回看。

**存储规则**：
- 最多保存 **10 条**记录
- 超过 10 条时，FIFO 删除最早的记录
- 用户点击「保存到历史」才保存（结果页有按钮）
- 未保存的决策不会自动记录

**历史页展示**：

| 字段 | 说明 |
|------|------|
| 时间戳 | 决策完成时间，格式 `2026-06-10 14:32` |
| 选项列表 | 如"火锅、烧烤、日料"（最多 3 个省略显示） |
| 推荐结果 | 最终推荐项名称 |
| 操作 | 「查看详情」→ 跳转结果页回显 |

**查看详情**：
- 复用结果页组件，传入历史数据重新渲染
- 回显时隐藏「保存到历史」按钮，显示「返回历史列表」

---

### 3.8 管理员功能（P2）

**功能描述**：提供问题库的完整 CRUD 管理。

**入口与鉴权**：
- 首页底部「⚙ 管理」入口
- 点击后弹出密码输入框
- 默认密码 `admin`（可在 config.json 中修改）
- 密码验证通过后进入管理页
- 连续 5 次密码错误，锁定入口 5 分钟

**问题库管理**：

| 操作 | 说明 |
|------|------|
| 查看列表 | 表格展示所有问题：序号、问题文本、权重、是否启用、操作 |
| 添加问题 | 输入问题文本 + 设置权重（1-3）+ 是否启用，点击添加 |
| 编辑问题 | 行内编辑或弹窗编辑问题文本、权重、启用状态 |
| 删除问题 | 确认弹窗后删除，附带"该问题关联的历史决策不受影响"说明 |
| 调整权重 | 滑块 1-3 倍，可拖动 |
| 启停用 | 开关控件，停用后该题不参与决策流程 |
| 保存修改 | 底部固定「保存修改」按钮，写入 questions.json |

**约束**：
- 问题文本不允许为空
- 问题文本不允许重复（trim 后比较）
- 至少保持 1 个启用的题目（删除到最后一个时给出警告）

---

### 3.9 导出功能（P2）

**功能描述**：将历史决策数据导出为文件。

**导出选项**：

| 格式 | 内容 | 文件名 |
|------|------|------|
| JSON | 完整的 history.json 数据结构 | `决策历史_2026-06-10.json` |
| CSV | 扁平化表格（每行=一次决策） | `决策历史_2026-06-10.csv` |

**CSV 列结构**：

```

```

**ZIP 导出**：「导出全部数据」将 questions.json + history.json + config.json 打包为 `数据备份_2026-06-10.zip`

## 5. 数据模型

### 5.1 questions.json — 问题库

```json
{
  "version": "1.0",
  "updatedAt": "2026-06-10T14:30:00+08:00",
  "questions": [
    {
      "id": "q001",
      "text": "你对价格敏感吗？",
      "weight": 3,
      "enabled": true,
      "createdAt": "2026-06-10T12:00:00+08:00"
    },
    {
      "id": "q002",
      "text": "口味对你来说有多重要？",
      "weight": 2,
      "enabled": true,
      "createdAt": "2026-06-10T12:00:00+08:00"
    },
    {
      "id": "q003",
```

## 6. 算法与逻辑

### 6.1 决策流程伪代码

```
function runDecision(options, enabledQuestions):
    // 1. 选取题目：最多 5 题，按权重降序
    selectedQuestions = enabledQuestions
        .sortBy(weight, desc)
        .take(maxQuestionsPerQuiz)   // 默认 5

    // 2. 用户逐题打分
    scores = {}  // { questionId: { option: score } }
    for each question in selectedQuestions:
        for each option in options:
            scores[question.id][option] = userInput(1..5)

    // 3. 计算加权总分
    finalScores = {}
    for each option in options:
        finalScores[option] = 0
        for each question in selectedQuestions:
            w = question.weight
            s = scores[question.id][option]
            finalScores[option] += w * s

    // 4. 选出最高分（平局取靠前的选项）
    recommendation = options.maxBy(o => finalScores[o])

    // 5. 生成推荐理由
    reason = generateReason(recommendation, finalScores, scores, selectedQuestions, options)

    // 6. 保存到历史（如果用户点击保存）
    saveToHistory({ options, scores, finalScores, recommendation, reason })

    return { recommendation, finalScores, reason }
```

### 6.2 题目选取逻辑

```
选取策略：
1. 从 questions.json 中筛选 enabled = true 的问题
2. 按 weight 降序排列
3. 取前 maxQuestionsPerQuiz（默认 5）道
4. 若启用问题数不足 5 道，则全部使用
```

### 6.3 分值归一化说明

统一采用 1-5 分制，不做归一化。每题权重直接乘原始分。原因：
- 所有题目共享同一个 1-5 量表，量纲一致
- 权重机制（1-3 倍）已足够表达重要性差异
- 保持计算逻辑透明，用户可自行验算

### 6.4 平局处理

当多个选项得分相同时：
1. 按选项输入顺序排列，取第一个
2. 在推荐理由中注明："【选项X】与【选项Y】得分相同（均为 XX 分），依输入顺序推荐【选项X】"
3. 柱状图中并列的选项用相同的高亮色标注

## 7. 推荐理由生成

### 7.1 规则模板引擎（默认）

**输入数据**：
- `winner`: 推荐选项名
- `runnerUp`: 第二名选项名
- `finalScores`: `{ option: score }`
- `scores`: `{ questionId: { option: score } }`
- `questions`: 题目列表（含权重和文本）

**生成逻辑**：

```
function generateReason(winner, runnerUp, finalScores, scores, questions):

    winnerScore = finalScores[winner]
    runnerUpScore = finalScores[runnerUp]
    gap = winnerScore - runnerUpScore
    gapPercent = gap / winnerScore

    // 句子 1：结论
    reason = "在 N 个选项中，【winner】综合得分最高（winnerScore 分）"

    // 句子 2：优势维度分析
    // 找 winner 得分最高（5 分优先，其次 4 分）且权重最高的题目
    bestDimension = findBestDimension(winner, scores, questions)
        // 规则：优先找 winner 得分 = 5 的题目
        //       其次找得分 = 4 的题目
        //       其中取权重最高的
    if bestDimension.score == 5:
        reason += "，尤其在「{questionText}」方面表现突出（5分）"
    else if bestDimension.score == 4:
        reason += "，在「{questionText}」方面表现较好（4分）"

    // 句子 3：与第二名的对比
    if gapPercent >= 0.1:  // 分差 ≥ 10%
        reason += "。相比第二名【runnerUp】有明显优势（领先 gap 分）"
    else:
        reason += "。与第二名【runnerUp】差距不大（仅差 gap 分），可凭直觉微调"

    // 句子 4（可选）：最高权重题目表现
    topWeightQuestion = questions.maxBy(q => q.weight)
    if scores[topWeightQuestion.id][winner]
       >= scores[topWeightQuestion.id].values().max():
        reason += "。且在你最看重的「{topWeightQuestion.text}」维度上也排名第一"

    return reason
```

## 12. 风险与边界条件

### 12.1 已知风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 问题库设计不合理导致推荐失真 | 用户不信任推荐结果 | 预设 5 道经过验证的默认题目；支持管理员随时调整 |
| 题目太多导致用户中途放弃 | 决策流程未完成 | 限制最多 5 题 + 显示进度条 + 每题默认值 3 可快速跳过 |
| localStorage 被清理导致数据丢失 | 历史记录丢失 | 支持 JSON/CSV/ZIP 导出；备份文件存磁盘 |
| AI 推荐理由调用失败 | 用户看到降级提示 | 降级为规则模板，体验不受影响 |
| 浏览器兼容性问题（尤其是 Firefox） | 部分用户无法使用 | 优先保证 Chrome/Edge；Firefox 允许轻微降级 |

### 12.2 边界条件

| 条件 | 行为 |
|------|------|
| 选项数 = 0 | 「开始决策」按钮置灰 |
| 选项数 = 1 | 「开始决策」按钮置灰，提示"至少输入 2 个选项" |
| 选项数 = 10（上限） | 正常处理，输入框下方提示"已达到上限（10 个）" |
| 选项数 > 10 | 弹出提示"最多支持 10 个选项，已截取前 10 个" |
| 启用问题数 = 0 | 「开始决策」按钮置灰，提示"暂无可用问题，请联系管理员" |
| 启用问题数 = 1 | 正常使用，仅 1 题 |
| 启用问题数 > 5 | 取权重最高的 5 题 |
| 所有选项得分完全相同 | 取第一个选项，理由中说明"各选项得分相同" |
| 管理员连续输错密码 5 次 | 弹出提示"已锁定，请 5 分钟后再试"，倒计时 |
| AI 请求超时 5 秒 | 降级为规则模板 |
