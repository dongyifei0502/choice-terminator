## 1. 产品概述

### 1.1 产品定位

「选择困难终结」是一款 **Web 全栈应用**（前端 SPA + Python Flask 后端 + SQLite 数据库），帮助用户在多个选项间通过多维度加权打分做出数据支撑的决策。

### 1.2 一句话描述

面对多个选项无法决定时，花 1 分钟走完提问打分流程，得到一个有理有据的推荐答案。

### 1.3 核心价值

| 痛点 | 解决方案 |
|------|----------|
| 选项太多，不知怎么比 | 系统引导从多个维度逐一打分，4 种场景预设覆盖常见决策 |
| 决策凭感觉，选完后悔 | 加权计分 + 推荐理由 + 柱状图，决策有数据支撑 |
| 反复纠结，浪费时间 | 1 分钟内完成，给出明确结论；同分随机选不卡住 |
| 数据只在一台设备 | 登录后历史云端同步，多设备共享 |

### 1.4 典型决策流程

```
打开网页 → 选场景 → 输入选项 → 逐题打分 → 查看结果 → 保存历史
```

## 3. 功能需求详述

### 3.1 场景选择（P0）

首页横向 pill 按钮组，4 种预设场景：

| 场景 ID | 标签 | 输入框 placeholder | 预设题目数 |
|---------|------|------|:---:|
| food | 🍽️ 餐饮 | 输入想吃的东西，如：火锅, 烧烤, 日料 | 5 |
| shopping | 🛒 购物 | 输入想买的东西，如：iPhone, 华为, 小米 | 5 |
| travel | ✈️ 旅行 | 输入想去的地方，如：成都, 大理, 三亚 | 5 |
| general | 📋 通用 | 输入选项，用逗号分隔，如：选项A, 选项B, 选项C | 5 |

切换场景时：placeholder 联动变化 + 问答页只从该场景选题。

---

### 3.2 选项输入（P0）

- 最少 2 个，最多 10 个
- 支持中英文逗号（`,` `，`）和空格分隔
- 实时解析为彩色标签（黄/粉/薄荷/蜜桃四色循环）
- 不足 2 个时"开始决策"按钮置灰
- 点击标签 ✕ 删除选项，输入框同步更新
- 连续分隔符不产生空标签

---

### 3.3 多轮提问与打分（P0）

- 选题逻辑：当前场景下启用的题目按 weight 降序，取前 5 题
- 每题展示：问题文本 + 权重 Badge（×1/×2/×3）+ 进度条
- 打分滑块：1-5 分，默认值 3，≥4 分数字框变黄
- 支持"上一题""下一题"导航，分数自动保存
- 最后一题按钮变为"✨ 查看结果"（粉色）

---

### 3.4 加权计算（P0）

```
选项X 最终得分 = Σ(问题i.weight × 选项X在问题i上的得分)
```

**示例**（3 选项，3 题）：

| 问题（权重） | 火锅 | 烧烤 | 沙拉 |
|-------------|:---:|:---:|:---:|
| 价格敏感吗？（×3） | 2 | 3 | 5 |
| 口味重要吗？（×2） | 5 | 4 | 2 |
| 环境重要吗？（×1） | 3 | 4 | 3 |

```
火锅 = 3×2+2×5+1×3 = 19
烧烤 = 3×3+2×4+1×4 = 21
沙拉 = 3×5+2×2+1×3 = 22 → 🏆 推荐沙拉
```

---

### 3.5 平局随机选择（P0）

- 同分时在并列第一中**随机**选 winner（不总是取靠前的）
- 结果页显示 🎲 标记 + "🎲 换一个"按钮
- "换一个"从平局选项中排除当前 winner 重新随机
- 推荐理由文案：`"各选项得分完全相同，🎲 随机推荐了【X】。同分的还有【Y】【Z】，可以点击换一个试试"`

---

| 元素 | 说明 |
|------|------|

规则模板引擎，详见 [第 7 节](#7-推荐理由生成)。不依赖外部 AI。

### 3.8 历史记录（P1）

- 存储策略：登录后云端优先 + localStorage 兜底；未登录纯本地
- 最多 10 条，超限淘汰最早
- 历史页展示：场景标签 + 时间 + 选项 + 推荐结果 + 🗑 单条删除
- 查看详情：复用结果页，隐藏保存按钮，显示返回按钮
- 清空 + 导出 JSON/CSV

---

### 3.9 用户系统（P2）

- 注册：用户名 2-20 字符 + 密码 ≥ 6 位
- 登录：JWT 令牌 72 小时过期
- 密码存储：SHA-256(固定盐) 哈希
- 认证：手写 HMAC-SHA256 JWT（零外部依赖）
- 首个注册用户或 username=admin 的注册自动获得 admin 角色
- 管理员默认账号：admin / admin123

| 操作 | 普通用户 | 管理员 |

| 管理系统题 | ❌ | ✅ |

- 表格显示：来源（系统/自定义）、场景、文本、权重、启用、操作

## 6. 算法与逻辑

### 6.1 加权计算

```
finalScores[option] = Σ(question.weight × scores[question.id][option])
```

### 6.2 平局处理

```
1. 找出 finalScores 中最高分 maxScore
2. 所有得分为 maxScore 的选项 → tiedOptions[]
3. 从 tiedOptions 中随机选一个 → winner
4. reroll(winner, tiedOptions):
   从 tiedOptions 中排除当前 winner，随机选新 winner
```

### 6.3 题目选取

```
1. 从 questions 筛选 enabled=true
2. 按 category 过滤（如果指定场景）
3. 按 weight 降序排列
4. 取前 5 题
```

## 7. 推荐理由生成

### 7.1 规则模板引擎

```
输入：winner, finalScores, scores, questions, options, isTie, tiedOptions

平局时：
  "在 N 个选项中，各选项得分完全相同（均为 X 分）。
   🎲 随机推荐了【winner】。同分的还有【Y】【Z】，
   可以点击换一个试试。"

非平局时：
  句1: "在 N 个选项中，【winner】综合得分最高（X 分）"
  句2: 找 winner 表现最好的维度
        - 5分优先：  "尤其在「维度名」方面表现突出（5分）"
        - 4分次之：  "在「维度名」方面表现较好（4分）"
  句3: 与第二名对比
        - gap% ≥ 10%: "相比第二名有明显优势（领先 X 分）"
        - gap% < 10%: "与第二名差距不大（仅差 X 分），可凭直觉微调"
  句4: 如 winner 在最高权重题也排第一 → 补一句
```

## 12. 边界条件

| 条件 | 行为 |
|------|------|
| 选项数 = 0 | 按钮置灰 |
| 选项数 = 1 | 按钮置灰，提示"至少输入 2 个" |
| 选项数 > 10 | 截取前 10 个 |
| 场景无启用题 | 回退到全部启用题 |
| 所有题都停用 | 提示"暂无可用问题" |
| 所有选项得分相同 | 随机选 + 换一个按钮 |
| 输入含 HTML 标签 | 转义显示不执行 |
| localStorage 不可用 | 降级为内存操作 |
| 被封禁用户登录 | 返回 403 |
| Token 过期 | 返回 401，需重新登录 |
| API 请求失败 | 前端降级本地存储 |
