## 第 6 页：最棒的 Prompt（对应答辩要求 3-展示1）

### Prompt 示例：Step 2 问答页

**Prompt**：
> "你现在是 Step 2：问答页的开发。依赖 Step 0（脚手架完成）和 Step 1（首页完成）。请实现：
> 1. 创建 js/router.js — 简易页面切换函数（display 显隐 + 触发目标页 init 函数）
> 2. 创建 js/services/questionService.js — selectQuestions() 按 enabled + weight 排序选题，loadQuestions() 三阶段加载（localStorage → fetch JSON → 兜底）
> 3. 创建 js/components/progressBar.js — updateProgress(current, total) 更新宽度+文字
> 4. 创建 js/components/slider.js — renderSlidersHTML() 渲染滑块（默认值3），bindSliderEvents() 实时更新分数（>=4 加 .high 类），collectScores() 收集分数对象
> 5. 修改 index.html 为多页面容器
> 6. 修改 app.js 整合路由 + 问答页逻辑
> 完成标准：首页输入3选项→进入问答→进度条+权重badge→滑块打分→前后导航→最后一题变粉色'查看结果'"

**为什么这个 Prompt 有效**：
1. **明确前置依赖** — "依赖 Step 0 和 Step 1"
2. **分模块编号** — 6 个子任务清晰独立
3. **指定函数签名** — `updateProgress(current, total)` 约束了入参类型
4. **附完成标准** — 可自行验证的验收条件
5. **上下文完整** — 单次会话能独立理解全部需求

## 第 7 页：AI 犯错案例（对应答辩要求 3-展示2）

### 案例 1：循环导入错误

**AI 写的代码**：
```python
# app.py
from backend.auth import auth_bp  # 导入 auth

# backend/auth.py
from app import get_db  # 又导入 app → 循环导入！
```

**发现过程**：
- 启动 `python app.py` → 报错 `ImportError: cannot import name 'auth_bp' from partially initialized module`
- 这是 Python 循环导入的经典错误

**修复 Prompt**：
> "出现了循环导入错误。auth.py 不能从 app 导入 get_db。请把 get_db 提取到独立的 backend/db.py 模块，让 app.py 和 auth.py 都从 backend.db 导入。"

### 案例 2：参数顺序错误

**AI 写的代码**：
```js
Store.setUser(data.user.username, data.user.role, data.token);
// 但函数签名是 setUser(token, username, role) ← 参数顺序反了！
```

**发现过程**：
- 登录后刷新页面 → 用户名显示的是 token 字符串
- 排查后确认是调用时参数传反了

**修复**：全局替换为 `Store.setUser(data.token, data.user.username, data.user.role)`

### 案例 3：依赖缺失

**AI 用了 flask-cors + bcrypt + pyjwt，但 PythonAnywhere 免费版无法安装**

**发现**：部署到 PythonAnywhere 后 HTTP 500，错误日志空白

**修复 Prompt**：
> "PythonAnywhere 免费版不能 pip install。请用纯标准库替代：hashlib.sha256 替代 bcrypt，hmac 替代 PyJWT，手写 after_request 替代 flask-cors。最终 requirements.txt 只需要 flask。"

## 第 8 页：Vibe Coding 使用反思（对应答辩要求 4）

### 好的方面

| 优势 | 说明 |
|------|------|
| 🚀 **效率极高** | 12 步前端 + 4 步后端 + 部署，1 天内完成，传统开发至少 1 周 |
| 📐 **结构清晰** | AI 严格按照开发计划的依赖关系执行，不会跳过依赖 |
| 🔧 **改起来快** | "四维改进"四个问题一次对话全部解决 |
| 🌐 **部署全程 AI** | 从 render.yaml 到 PythonAnywhere API 上传文件，全部 AI 操作 |

### 需要改进的方面

| 不足 | 说明 |
|------|------|
| ⚠️ **AI 幻觉** | AI 会使用不存在的 API（如 PythonAnywhere console API）需要人工验证 |
| 🧪 **测试依赖人工** | AI 无法自动打开浏览器测试，防御阶段的验证全靠人 |
| 📝 **Prompt 质量决定结果** | 功能描述不清时 AI 容易写出"看起来对但实际不完整"的代码 |
| 🔐 **安全需人工审查** | AI 写的认证/加密代码需要人工确认逻辑正确性 |

### 核心体会

> **Vibe Coding ≠ 不思考。恰恰相反，它要求更强的「架构思维」和「Prompt 设计能力」。**
>
> 你不需要写代码，但你必须：
> 1. 把大问题拆成小步骤（分模块、分步骤）
> 2. 每一步清晰定义输入、输出、边界条件
> 3. 亲自验证每一步的输出是否正确
> 4. 对最终代码负 100% 责任
